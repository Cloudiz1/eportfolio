Spotacent 2: Source

This project began with a small issue. I had realized that Spotify does not support playing music while offline unless the user has purchased their premium plan. Unfortunately, plane rides do not come bundled with free internet nor Spotify premium. My solution? Build an entire music player from the ground up.

The project found on github is actually my second attempt at this. Originally, I had opted for a visual GUI. After deciding my code was inadequate, I decided to restart and opt for a command line application instead.

Spotacent supports a number of features such as creating playlists, adding songs, changing settings (volume, looping, shuffling), skipping, pausing, unpausing, and more. It works by extracting the audio of a YouTube video into a .wav file. From there, it uses the python module Pygame to play the audio files. This project posed a number of unique challenges and taught me threading as well as building command line interfaces.

Simple Bootup Script: Source

This project idea came as a result of a small inconvienence. I commonly use Spotify to play music and Discord to communicate with my friends. Although both of these apps are great on their own, I prefer to modify them with third-party extensions. I used both Spicetify and BetterDiscord, which both provides their respective applications with a marketplace and API. I would often have to reinject or reinstall the extension entirely. As a result of this, I thought I should make a script to do that on startup.

Spicetify was easy. Their CLI offers a built in command to update and inject Spicetify. All I had to do was run a command on terminal. BetterDiscord, however, was less simple. The solution I came up with was to check the github repository for an update via web scraping. If a new version was detected, I would run the installation script. Unfortunately, I had presumed each time Discord launched without the third-party extention meant that Discord had updated and subsequently broken BetterDiscord. This does not appear to be the case as the github repository updates infrequently. Despite this oversight, I don't regret this project as it taught me web scraping, parsing data, and more

Notes App: Source

I originally made this to take digital notes more efficiently. I was originally inspired by this video where a math major with the online alias "SeniorMars" demonstrates how to take notes with Vim and LaTeX. Around this time, my fall classes had just started and I wanted to try online notes for once. With this motivation, I created this application which has syntax similiar to HTML. Moreover, it generates a table of contents and supports images, lists, LaTeX, and more.

To take notes, one simply edits a .txt file. On the backend, a lexer first parses the file into managable tokens. These tokens generate HTML elements through a simple switch case statement. A list keeps track of major headers and subheaders and their ids. Using this list, a simple table of contents can be generated.

Tasks App: Source

I would consider this simple task list the first project I made. At the time, I thought a simple application to communicate between frontend and backend would teach me much more about node.js and web development as a whole. On app startup, all tasks with the proper information will be displayed with each task having a respective card. Adding a task is as simple as pressing a button in the top left corner and filling out a few fields. On the backend, it stores the tasks in a .json file and will read and write to it when necessary.
